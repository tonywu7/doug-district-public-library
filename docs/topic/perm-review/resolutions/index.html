  • Explain better the different ways to contact the mods 
    • Contacting the mod team

Read the FAQ first: https://www.dougdougw.com/faq

- For questions and issues that do not need to be resolved immediately, open a ticket by pasting the following command in any channel: ?ticket open new ticket
- For issues that require attention, but not urgently, ping one Discord mod individually, most of the time mods can respond in a timely manner
- For urgent problems (e.g. server raids or gross rule violations), ping the @Discord Mods role.

Low disruption changes

These changes affect role permissions. However, they are expected to not change any existing server functions, and even if functions are affected, they will be easy to fix.

Bot hierarchy patch

  • Deprecate @Functional Bots and @Hidden Bots
  • New role @Admin Bots
    • Candidate bots:
      • @Clarence @Frank @Fondle
    • Precedence (high to low):
      • @Lord Cockswain Jr → @Admins → @Admin Bots → @MEE6 → @YAGPDB.xyz → @AltDentifier
  • New role @Utility Bots
    • These bots are meant to be used by the community, and only in specific channels (e.g. VC, except Frank but Frank is covered by admin already)
    • Candidate bots:
      • @Amanda (Team Rocket) @Farkas @Frank
    • Access: use this role to limit integration bots to the channel where they operate (in VCs)
    • Precedence (high to low):
      • @Discord Mods → @Twitch Mods → @Utility Bots → @Rythm → @Groovy
  • New role @Integration Bots
    • These bots are for integrations: webhooks, subscriptions, Minecrafts, etc. They are likely to be hidden from members list (i.e. not listed separately)
    • Candidate bots:
      • @Zapier @Patreon @DougBot
    • Access: use this role to limit integration bots to the channel where they operate (in #content, Minecraft channels etc.)
    • Precedence (high to low):
      • @Groovy → @Integration Bots → @Zapier → @Patreon Bot → @DiscordSRV
  • Modify role @Bots
    • This role grants basic message permissions and perms often used by bots
    • Candidate bots: all bots
    • Access: this role should not be used to control channel access (meaning no channel override with this role having view_channels allow or view_channels deny
    • Precedence (high to low):
      • @Dannyling → @Bots
